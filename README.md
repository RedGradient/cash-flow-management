# Cash Flow Management

Демо-проект для управления денежными потоками на Django. Позволяет создавать, читать, обновлять и удалять транзакции с категориями и подкатегориями через браузер.

## Стек

- Python 3.12
- Django 6
- Django REST Framework
- PostgreSQL 16
- Docker / Docker Compose

## Возможности

- Веб-интерфейс для работы с данными в браузере
- Логирование действий и событий приложения
- Транзакции с суммой, типом (пополнение / списание), статусом и категорией
- Настроенная админ-панель Django для управления данными

## Быстрый запуск (docker compose)

```bash
make up
```

Команда создаёт `.env` из `.env.example` (если файл ещё не существует) и запускает приложение и PostgreSQL.

После запуска:

- приложение: http://127.0.0.1:8000/
- админка: http://127.0.0.1:8000/admin/

Суперпользователь создается автоматически при первом запуске (логин: admin, пароль: admin)

## Переменные окружения

Скопируй `.env.example` в `.env` и при необходимости измени значения:

| Переменная | Описание |
|------------|----------|
| `SECRET_KEY` | Секретный ключ Django |
| `DEBUG` | Режим отладки (`True` / `False`) |
| `ALLOWED_HOSTS` | Разрешённые хосты, через запятую |
| `POSTGRES_DB` | Имя базы данных |
| `POSTGRES_USER` | Пользователь PostgreSQL |
| `POSTGRES_PASSWORD` | Пароль PostgreSQL |
| `POSTGRES_HOST` | Хост БД (`localhost` локально, в Docker подставляется `postgres`) |
| `POSTGRES_PORT` | Порт PostgreSQL |

## Модели

### Category

Двухуровневая иерархия категорий:

```
Списание (Withdrawal)
├── Маркетинг
│   ├── Farpost
│   └── Avito
└── Инфраструктура
    ├── VPS
    └── Proxy
```

Корневая категория имеет тип (`Deposit` / `Withdrawal`). Подкатегория тип не хранит — он наследуется от родителя.

### Transaction

| Поле | Описание |
|------|----------|
| `amount` | Сумма |
| `type` | Тип: пополнение или списание |
| `status` | Статус: Business / Personal / Tax |
| `category` | Категория или подкатегория |
| `comment` | Комментарий (необязательно, до 100 символов) |
| `created_at` | Дата создания (автоматически) |
